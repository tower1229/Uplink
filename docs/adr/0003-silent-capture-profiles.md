# Capture Profiles are silent Repository data

Capture calibration may feel like an in-extension self-correction UX, but the resulting declarative rules are still persisted in the Repository as Capture Profiles so backup and device migration keep working. They are not exposed as a first-class user product (no profile gallery or required CLI). Extension-only storage was rejected because it drops rules on reinstall; a fully explicit Profile product was rejected as unnecessary surface area.
